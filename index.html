Case 1

다해줌 대상 쇼핑몰이 전체 최저가

가장 기본적인 노출 방식입니다. 최저가 영역과 몰별최저가 영역 모두에서 다해줌을 바로 인지할 수 있어 사용자의 선택 유도가 가장 직관적입니다.

맨 위로
예시 화면

특징

  • 최저가 영역 우측에 다해줌 버튼 노출
  • 몰별최저가 영역에서도 다해줌 업체 버튼 노출
  • 가격 경쟁력과 다해줌 진입 경로가 동시에 명확하게 보임

유의사항

  • 최저가 강조와 다해줌 CTA가 동시에 보이므로 시각적 우선순위 충돌이 없도록 정리해야 함
  • 다른 배지·할인 문구가 있는 경우 버튼 인지가 약해지지 않도록 여백과 색 대비를 확보해야 함
Case 2

멤버십/카드 혜택가가 최저가이면서 다해줌 업체

일반 판매가가 아니라 혜택가 맥락에서 다해줌이 우위인 케이스입니다. 가격 오인을 피하려면 혜택 적용 조건과 노출 위치가 함께 설계되어야 합니다.

맨 위로
예시 화면

특징

  • 최저가 영역 우측에는 다해줌 버튼 비노출
  • 멤버십/카드 혜택가 영역에 다해줌 버튼 노출
  • 몰별최저가 영역에는 다해줌 업체 버튼 노출

유의사항

  • 멤버십 포인트는 실제 결제자인 다해줌이 취득하므로 향후 다른 구매대행에 활용 가능
  • 카드 할인은 횟수 제한 또는 최대 할인 금액 제한이 있어, 실제 할인 금액에 살 수 없을 수 있음
Case 3

다해줌 대상 쇼핑몰이 전체 최저가가 아닌 경우

전체 최저가가 아닌 상황에서 다해줌을 보조 동선으로만 노출하는 방식입니다. 가격 신뢰성을 해치지 않으면서도 다해줌 유입 경로를 유지할 수 있습니다.

맨 위로
예시 화면

특징

  • 최저가 우측 영역에는 다해줌 버튼 비노출
  • 몰별최저가 영역에서만 다해줌 업체 버튼 노출
  • 전체 최저가 정보와 다해줌 선택 유도를 분리해 오인을 줄일 수 있음

유의사항

  • 전체 최저가가 아님에도 과도한 강조를 하면 사용자가 추천 이유를 혼동할 수 있음
  • 목록 내 버튼만 보이므로 식별성과 클릭 유도성이 지나치게 약해지지 않도록 대비를 확보해야 함
Case 4

최저가가 아닌경우 다해줌 기준 최저가를 별도 영역으로 보여주는 확장안

전체 최저가는 아니지만 다해줌 기준의 최저가를 별도 영역으로 제시하는 확장형 패턴입니다. 설명력은 높아지지만 정보 구조를 더 정교하게 다뤄야 합니다.

맨 위로
예시 화면

특징

  • 신규 영역을 추가하여 다해줌 기준 최저가 및 다해줌 버튼 표시
  • 몰별최저가 영역에서도 다해줌 업체 버튼 노출
  • 전체 최저가와 다해줌 추천가를 병렬로 비교할 수 있음

유의사항

  • 전체 최저가와 다해줌 기준 최저가를 시각적으로 명확히 구분하지 않으면 혼동이 발생할 수 있음
  • 추가 영역이 생기므로 모바일에서는 정보량이 과해지지 않도록 접힘 구조나 요약 문구 설계가 중요함
  • 왜 다해줌 가격을 별도로 제안하는지 설명하는 보조 문구가 함께 있어야 설득력이 높아짐
공통 정책

가격비교 탭 공통 노출

5페이지 기준으로 가격비교 탭에서는 다해줌 상품에 대해 다해줌 버튼이 공통적으로 노출됩니다. 즉, 1~4번 케이스는 메인 가격 영역의 노출 방식 차이를 설명하는 것이고, 가격비교 탭의 다해줌 버튼 노출은 모든 케이스에 동일하게 적용되는 공통 기준입니다.

맨 위로
공통 정책 예시 화면 5페이지 기준 이미지

핵심 포인트

  • 가격비교 탭에서는 다해줌 상품에 대해 다해줌 버튼이 공통 노출됩니다.
  • 이 정책은 1~4번 케이스와 별개로 동일하게 적용되는 공통 규칙입니다.
  • 사용자는 가격 영역 케이스가 달라도 가격비교 탭에서는 일관된 진입 경험을 얻게 됩니다.
